Homelab Infrastructure

Active

This website is hosted from my personal homelab using a secure reverse-tunnel architecture. The setup avoids public IP exposure and traditional port forwarding, while still providing global HTTPS access.

Live Request Path

Browser
Cloudflare Edge

Portfolio

Edge Layer

  • Cloudflare handles DNS and HTTPS termination.
  • No direct exposure of home network.
  • Traffic proxied securely via tunnel.

Networking

  • ISP uses dynamic IP (no static IP).
  • Port forwarding intentionally avoided.
  • Cloudflare Tunnel bypasses CGNAT limitations.

Compute Layer

  • Fedora-based home server.
  • Nginx serving static content.
  • SELinux configured for secure access.

Service Exposure

  • cloudflared runs as a systemd service.
  • Maps public domain → localhost:80.
  • Always-on secure outbound tunnel.

Request Flow (Simplified)

  1. User requests chinmayk.dev
  2. Cloudflare resolves DNS and handles HTTPS
  3. Request routed through Cloudflare Tunnel
  4. Local cloudflared forwards to Nginx
  5. Nginx serves static portfolio files

Design Decisions:

  • Avoided exposing home network via port forwarding.
  • Used reverse tunnel architecture for reliability.
  • Offloaded TLS/SSL to Cloudflare edge.

Current Build Stage: Moving this setup toward a containerized platform with CI/CD automation, service-level monitoring, and environment isolation.
